# obsidian-jsx-renderer

## 目的

ClaudeがJSXアーティファクトとして出力したインタラクティブUIを、Obsidian上でそのままレンダリングするプラグイン。

## ワークフロー

1. Claude（claude.ai等）がJSXアーティファクトを生成
2. Obsidian MCP経由でVaultに `.jsx` ファイルまたはマークダウンのコードブロックとして保存
3. このプラグインがObsidian上でReactコンポーネントとしてレンダリング

## 機能要件

### コードブロックレンダリング
- マークダウン内の ` ```jsx ``` ` コードブロックをインタラクティブなReactコンポーネントとしてレンダリング
- 小さなコンポーネントの埋め込みに使用

### .jsx ファイルビュー
- `.jsx` ファイルをObsidianで開くと、ソースではなくレンダリング結果を表示
- 大きなコンポーネント（レシピアプリ等）はファイルとして保存→開く

## 技術設計

### JSXトランスパイル
- **Sucrase** を使用（軽量・高速、~100KB）
- ランタイムでJSX → JSに変換
- Babelは重すぎるので不採用

### React
- React + ReactDOM をプラグインにバンドル
- `import { useState } from "react"` 等のimportをインターセプトして内蔵Reactを返す

### レンダリングの仕組み
1. Sucrase で JSX → JS に変換（`imports` + `jsx` transform）
2. `require("react")` をモックして内蔵Reactを返す
3. `new Function()` で評価し、`export default` されたコンポーネントを取得
4. `createRoot()` でコンテナにレンダリング

### セキュリティ
- `new Function()` による動的コード実行を使用
- Vault内のコードのみ実行されるため、個人利用では許容範囲
- 将来的にサンドボックス化を検討してもよい

## 技術スタック

- TypeScript
- React 18 + ReactDOM
- Sucrase（JSXトランスパイラ）
- esbuild（バンドラ）
- Obsidian Plugin API

## 参考：対象となるJSXの例

Claudeが生成するレシピナビアプリ（`nikumiso-cabbage-v2.jsx`）のようなコンポーネント:
- `useState` 等のReact Hooksを使用
- `export default function Component()` 形式

- 外部依存なし（React本体のみ）

- CM6対応が壊れており最新Obsidianで動作しない
- フォークするよりスコープを絞って新規作成する方が合理的と判断
